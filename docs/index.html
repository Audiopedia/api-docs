audiopedia.app API Reference

This is the API documentation for the audiopedia.app.

If you wish to run the interactive GraphIQL tool or try out the queries in the docs, first start the API server using python manage.py runserver.

There are also example uses of each of these queries/mutations in api/audiopedia/test_audios.py.

Note: The "object" type that you see in the docs refer to the graphene.ID type. Read more about it here.

API Endpoints
API endpoint:
http://127.0.0.1:8000/graphql/
Version: 1.0.0

Authentication

Queries and mutations for authentication.

Note: To authenticate with tokens and login, we have to pass the tokenAuth mutation along with an Authorization HTTP header that contains the token prefixed by JWT.

For more information take a look at the bottom of this page.

ID: object

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

String: string

The String scalar type represents textual data, represented as UTF-8 character sequences. The String type is most often used by GraphQL to represent free-form human-readable text.

TopicInput: object

index:
ID

Position/placement of the topic among a list of topics

title:
String

Title of the topic

audioUrl:
String

URL to the audio directory associated with this topic

active:
Boolean

Inactivate to temporarily delete topic and reactivate to recover

published:
Boolean

Visible to user if true

playlists:
ID

List of all the IDs of playlists this topic contains

Example
{
  "index": "object",
  "title": "string",
  "audioUrl": "string",
  "active": "boolean",
  "published": "boolean",
  "playlists": [
    "object"
  ]
}
